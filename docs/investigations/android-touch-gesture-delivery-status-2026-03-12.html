Android Touch Gesture Delivery Status

Current conclusion: the previous Android touch parity work was insufficient. It added diagnostics and removed one old native swallow path, but real-device testing still shows touch is effectively blocked in the lyric scene. The strongest remaining suspect is now the WebView/browser suppression stack, not the old single native gate alone.

What is proven now

  • MainActivity forwards touch and logs native delivery decisions.
  • The browser still has lyric-scene touchstart/touchmove handlers and logs receipt.
  • Mouse works, touch does not.
  • This strongly suggests the shared interaction model is not the first blocker.

Why the last pass failed

Insufficient

The work changed delivery and logging, but it never isolated whether WebView/browser suppression was the real culprit. So it improved observability without proving the cause of the real-device failure.

Strongest current culprit

  • Injected viewport lock with user-scalable=no
  • Injected document.body.style.touchAction = 'none'
  • Injected userSelect / webkitTouchCallout suppression
  • #lyric-display also using touch-action: none

Any one of these might be acceptable alone. The current stack may simply be too aggressive in combination.

Would more iterations help?

Blind rewrites: low value Another broad fix pass without isolation is likely to waste time again.

One A/B experiment: high value Add a flag that disables the current browser/WebView suppression layer and test one APK on a real device.

Recommended next move

  1. Add one runtime/build flag that disables the current WebView/browser touch suppression block.
  2. Build one APK with the flag flipped.
  3. If touch comes back, the culprit is confirmed.
  4. Re-enable protections one piece at a time to find the minimal safe subset.
  5. Only after that, revisit any remaining browser-side binding issues.

Bottom line

The previous touch work was not final. The best next step is not another guess-driven rewrite. It is one explicit flag-based A/B experiment against the suppression layer, because that is the strongest remaining blocker in the actual source.
